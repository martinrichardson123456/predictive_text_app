Project description

  • Create a predictive text application suitable for use on a mobile phone.
  • Must not occupy too much storage space or have too slow an algorithm.
  • Given a string of input characters, predict the next word / end of current word.
  • Training texts used for predictions are (US English) blogs and news articles.
  • Twitter posts also provided but are not used due to abundance of spelling errors etc.
  • N-grams (i.e. word sequences, N = 1, 2, 3 and 4) and number of times each appears are extracted from text corpora.
  • Rather than use small random samples of corpora, entirety is processed in chunks, e.g.:

Algorithm description

  • N-gram libraries organised alphabetically; one file for 1-grams that begin with 'aa...', ab...', ac...', all the way up to 'zz...' (i.e. file names 'aa.txt', 'ab.txt', etc.). 2, 3, and 4-gram libraries organised similarly but by first 3 letters rather than first 2 letters.
  • The user's typed input is truncated to a maximum of the 4 most recently-typed terms.
  • The first 2 or 3 characters in this sequence then dictates which file is scanned:

User has entered "that is". 2-gram file "tha.txt" opens; subset with 'term1 = that' is found.

Within subset from previous step, subset with 'term2 = is' is found.

The number of times each 2-gram appears in the copora.

The number of unique 3-grams each 2-gram begins (needed for Kneser-Ney calc.).

The total number of 3-grams each 2-gram begins (needed for Kneser-Ney calc.).

  • After this filtering process is complete, a (user-specified) number of the most probable suggestions for the final term are returned.
  • If this search returns no suggestions, a Kneser-Ney back-off algorithm is initiated, involving finding the most probable suggestions based on lower-order N-gram searches.

  • Algorithm tested using Google N-gram files; 1000 tests for each order of N-gram.
  • Testing proceeds letter-by-letter through final term until final term is suggested.
  • 4-gram search time <1.0 s (<1.5 s) in 92.0% (99.0%) of cases; ~3-5 MB memory usage.
  • ~91%-92% of searches successfully find the intended N-gram in the libraries.
  • ~52%-72% of intended terms suggested before 50% of term has been typed.
  • 4-gram library mean (max) file size on disk: 5.1 KB (5.6 MB). Total N-gram libraries: 344 MB (85 MB zipped).
